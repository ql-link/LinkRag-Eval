# Query 软分流候选方案

## 状态

本文记录下一阶段待验证的 Query 软分流候选参数。它们是实验起点，不是已经验证的最优参数，也不替代当前默认 Hybrid 配置。

当前生产候选基线仍为三路 `weighted_score`：Dense/Sparse/BM25 权重 `0.70/0.15/0.15`，分路 TopK `150/50/100`，阈值 `0.30/0.20/0`，最终 TopK `10`。

## 候选规则

| Query 特征 | Dense | Sparse | BM25 | 额外处理 |
| --- | ---: | ---: | ---: | --- |
| 短关键词 | 0.45 | 0.40 | 0.15 | 保护 Sparse Top3 |
| 明确编号/日期/版本号 | 0.30 | 0.35 | 0.35 | 精确词命中加分 |
| 长描述/多条件 | 0.45 | 0.40 | 0.15 | 去停用词并提取关键条件 |
| 自然语言改写 | 0.70 | 0.20 | 0.10 | 保护 Dense Top4 |
| 无明显特征 | 0.70 | 0.15 | 0.15 | 使用当前默认 Hybrid |

## 验证纪律

1. 只使用 balanced tune 420 条样本识别规则、搜索参数和比较收益。
2. 当前 balanced blind 180 条已经揭盲，不得继续用于选择权重、保护名额或分类阈值。
3. 每条 Query 最多命中一个主类别；无法高置信分类时回退默认 Hybrid。
4. 候选保护必须按 `chunk_id` 去重，保护位与加权融合结果共同组成最终 Top10。
5. 以 chunk 粒度 Recall@10 为主目标，MRR 为次目标；同时报告各场景和整体结果。
6. 规则冻结后，使用新建 blind v2 或下一批真实 Query 只验收一次。

## 待测对照组

1. 当前默认 Hybrid 基线。
2. 仅按 Query 特征动态调整权重。
3. 动态权重加 Sparse/Dense 候选保护。
4. Query 特征加召回结果置信度的联合软分流。
