# Query 软分流候选方案

## 状态

本文同时记录早期动态权重候选和 2026-07-20 完成的候选深度优化。动态权重与候选保护仍是实验方案；候选深度已经只用 Tune 冻结，并完成未曝光 Blind v3 离线验收，但尚未替代生产默认 Hybrid。

当前生产候选基线仍为三路 `weighted_score`：Dense/Sparse/BM25 权重 `0.70/0.15/0.15`，分路 TopK `150/50/100`，阈值 `0.30/0.20/0`，最终 TopK `10`。

## 候选深度验证（2026-07-20）

本轮目标是先提高“正确 Chunk 是否进入三路候选并集”的上限。选参只使用 2,000 条 Tune；Blind v2 已揭盲，只在参数冻结后做回归观察。

| 方案 | Tune 候选覆盖 | 平均理论候选预算 | Blind v2 回归覆盖 |
| --- | ---: | ---: | ---: |
| 原基线 `150/50/100` | 95.25% | 300 | 89.05% |
| 无分流回退 `200/50/200` | 98.25% | 450 | 96.19% |
| 冻结 Query 分流 | **98.55%** | **437.83** | **98.10%** |

冻结深度如下。分类器只读取运行时 Query 文本，不读取场景标签或 qrel：

| Query 桶 | Dense | Sparse | BM25 | Tune 覆盖 |
| --- | ---: | ---: | ---: | ---: |
| 短关键词 | 300 | 100 | 225 | 98.17% |
| 明确编号/日期/版本号 | 150 | 50 | 100 | 100.00% |
| 数字/时间条件 | 275 | 50 | 200 | 98.47% |
| 长描述/多条件 | 125 | 50 | 75 | 99.12% |
| 自然语言/默认 | 150 | 50 | 225 | 98.38% |

`exact_identifier` 桶在当前 Tune 中没有“场景标签和文本门禁同时有效”的代表样本，因此保守保留原深度，不采用网格给出的激进降深。阈值继续使用 `0.30/0.20/0`，与零阈值候选覆盖相同。

候选保护不能补回未进入候选池的 Chunk，本轮不新增保护位。编号精确匹配继续由 LambdaMART 的 `identifier_exact_coverage` 特征处理，不在无有效样本时人工设置加分系数。完整报告：
`runs/golden_v2/scale_100k_991004/scale_20k_overnight/candidate_coverage_optimization_20260720/candidate_coverage_optimization_report.html`。

注意：98.55% 是候选覆盖率，不是 Recall@10。完整深度缓存、Hybrid/LambdaMART Top10 和未曝光 Blind v3 已于同日完成，结果见下节。

## 冻结 Top10 与 Blind v3 结果（2026-07-20）

2,000 条 Tune 全量分流缓存无失败，实际去重候选数均值 278.56、中位数 276。只使用 Tune OOF
冻结 `n_estimators=31`、`learning_rate=0.03`、`blend_alpha=1.0`、`protect_baseline_top_k=0`。

| 数据集 | n | 候选覆盖 | 分流 Hybrid Recall@10 | LambdaMART Recall@10 | 变化 |
| --- | ---: | ---: | ---: | ---: | ---: |
| Tune OOF | 2000 | 98.55% | 35.75% | 44.90% | **+9.15pp** |
| 未曝光 Blind v3 | 150 | 92.67% | 22.67% | 30.67% | **+8.00pp** |

Blind v3 在参数冻结后只运行一次，Query 与正证据的历史重叠均为 0。五个场景各 30 条：

| 场景 | 候选覆盖 | Hybrid | LambdaMART | 变化 |
| --- | ---: | ---: | ---: | ---: |
| 相似文档 | 100.00% | 30.00% | 53.33% | **+23.33pp** |
| 多条件 | 93.33% | 46.67% | 60.00% | **+13.33pp** |
| 别名 | 96.67% | 10.00% | 16.67% | +6.67pp |
| 自然语言改写 | 93.33% | 20.00% | 20.00% | 0.00pp |
| 短关键词 | 80.00% | 6.67% | 3.33% | **-3.33pp** |

结论：候选分流与 LambdaMART 的整体收益已获得独立验证，但绝对 Recall@10 仍低，短关键词必须增加
低置信度回退 Hybrid 的生产门禁。当前 20k 语料没有可构造严格编号、日期、版本号 Blind 题目的未曝光证据，
因此 Blind v3 没有伪造这两个场景；它们需要先补充 eval-only 语料后单独验收。

| Query 特征 | Dense | Sparse | BM25 | 额外处理 |

## 验证纪律

1. 候选深度和 LambdaMART 最终参数只使用扩展 Tune 2000 条样本识别规则、搜索参数和比较收益。
2. 当前 Blind v2 与 Blind v3 都已经揭盲，不得继续用于选择权重、保护名额或分类阈值。
3. 每条 Query 最多命中一个主类别；无法高置信分类时回退默认 Hybrid。
4. 候选保护必须按 `chunk_id` 去重，保护位与加权融合结果共同组成最终 Top10。
5. 以 chunk 粒度 Recall@10 为主目标，MRR 为次目标；同时报告各场景和整体结果。
6. 规则冻结后，使用下一批真实 Query 或新建 Blind v4 只验收一次。
